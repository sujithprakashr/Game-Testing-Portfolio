# 🎮 Level Devil – Game Testing Project

This folder contains structured manual testing documentation for the Level Devil mobile game.  
Testing activities were managed using Jira for issue tracking and Xray for test case management and execution.

---

## 📌 Project Overview

The objective of this project was to validate core gameplay mechanics, unexpected trap behaviors, and overall system stability through structured test design and execution.

---

## 🔍 Scope Covered

- Game Launch & Initial Load
- Level Navigation & Progression
- Character Movement & Controls
- Collision Detection & Physics Behavior
- Obstacles & Hidden Trap Mechanics
- Retry / Restart Functionality
- UI Behavior Across Levels
- Edge Case & Negative Scenarios

---

## 🎭 Troll & Unexpected Mechanics Validation

Level Devil is intentionally designed with unpredictable “troll” elements such as:

- Sudden platform disappearance
- Hidden spikes and traps
- Fake safe zones
- Unexpected obstacle movement

Testing focused on:

- Verifying that troll mechanics trigger consistently
- Ensuring traps behave as designed (not as unintended bugs)
- Confirming restart logic functions correctly after unexpected failures
- Validating that difficulty spikes are intentional and stable

---

## 🧪 Testing Types Performed

- Functional Testing
- Gameplay Flow Validation
- Exploratory Testing
- UI Testing
- Regression Testing
- Edge Case Validation

---

## 🛠 Tools Used

- Jira (Issue Tracking)
- Xray (Test Case Management & Execution)

---

## 📊 Outcome

- Validated core gameplay and troll mechanics behavior
- Confirmed consistent trap execution across test cycles
- Ensured stable restart and retry logic
- No unintended critical defects observed during structured testing
