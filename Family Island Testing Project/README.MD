# 🏝️ Family Island – Mobile Game Testing Project

This folder contains structured manual testing documentation for the Family Island mobile game.  
Testing activities were managed using TestRail for test case management and execution tracking.

---

## 📌 Project Overview

The objective of this project was to validate core gameplay mechanics, resource management systems, progression flow, and overall application stability through structured test planning and execution.

---

## 🔍 Scope Covered

- App Launch & Login
- Core Gameplay Mechanics
- Energy Consumption & Recovery Flow
- Resource Collection & Management
- Crafting System Validation
- Quest Completion Flow
- Island Exploration
- UI/UX Consistency
- Interruptions Handling (Incoming calls, background/foreground transitions)
- Stability Testing

---

## 🎮 Gameplay & System Validation

Testing focused on:

- Verifying smooth progression across levels and quests
- Ensuring correct energy deduction and regeneration logic
- Validating crafting completion and reward distribution
- Confirming resource updates reflect accurately in inventory
- Testing interruption recovery without data loss
- Ensuring stable performance during extended gameplay sessions

---

## 🧪 Testing Types Performed

- Functional Testing
- Gameplay Flow Validation
- Regression Testing
- Exploratory Testing
- UI Testing
- Stability Testing
- Edge Case Validation

---

## 🛠 Tool Used

- TestRail (Test Case Management & Execution Tracking)

---

## 📊 Outcome

- Validated end-to-end gameplay flow
- Confirmed correct resource and energy calculations
- Ensured crafting and quest systems functioned as expected
- Verified application stability during multiple gameplay cycles
- No critical functional defects observed during structured testing
